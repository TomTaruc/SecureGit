## BUG 1 — Collaborators don't see their repos on the dashboard

**Severity:** High
**Root cause confirmed in:** `securegit-backend/app/routes/dashboard.py`

The `/api/dashboard/stats` endpoint computes `total_projects` by checking
`Collaborator.query.filter_by(user_id=user_id)` — that part is correct. **But**
`securegit-frontend/src/pages/DashboardPage.jsx` and `ProjectsPage.jsx` both render from
`projectsApi.listProjects()`, which calls `GET /api/projects`. That route
(`securegit-backend/app/routes/projects.py`, `list_projects()`) **is already correct**:

```python
owned = Project.query.filter(Project.owner_user_id == user_id, Project.deleted_at.is_(None)).all()
collaborated_ids = [c.project_id for c in Collaborator.query.filter_by(user_id=user_id).all()]
collaborated = Project.query.filter(
    Project.project_id.in_(collaborated_ids),
    Project.owner_user_id != user_id,
    Project.deleted_at.is_(None)
).all()
all_projects = owned + collaborated
```

This logic is sound. **The actual defect is in `require_project_access` permission resolution**,
not visibility. Look at `securegit-backend/app/utils/rbac.py` → `check_permission()`:

```python
def check_permission(user: User, project: Project, perm: str) -> bool:
    if user.role == "admin":
        return True
    if project.owner_user_id == user.user_id:
        return True
    collab = get_actor_permission(user, project)
    if collab is None:
        return False
    return collab.has_permission(perm)
```
